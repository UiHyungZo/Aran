# UI Guide

## 목적
AI가 기본 스타일로 임의 구현하지 않도록 Aran 앱의 화면 톤, 컴포넌트 패턴, 금지 패턴을 정의한다.

## 기본 원칙
- 의료/치료 기록 앱이므로 차분하고 명확한 UI를 우선한다.
- 정보 계층을 분명히 하고, 장식보다 가독성을 우선한다.
- Empty, Loading, Error 상태를 화면별로 명시한다.

## 컴포넌트 규칙
- 치료 일정, 약물, 검사 기록은 카드 기반 리스트를 우선한다.
- 주요 CTA는 화면당 1개를 원칙으로 한다.
- 위험/삭제 액션은 확인 단계를 둔다.

## 금지 패턴
- 과도한 glass morphism
- 의미 없는 그라데이션 텍스트
- 의료 정보 화면에서 과도한 애니메이션
- ViewController/View 내부에서 임의 색상 하드코딩 남발
